## Архитектура приложения
Слои архитектуры образуют шаблон проектирования Model-View-Presenter, сокращённо MVP:
- слой представлений, отвечает за отображение данных на странице, 
- слой данных, отвечает за хранение и Изменение данных,
- презентер, отвечает за связь представления и данных.


### Базовый код 
#### Класс Api 
Содержит в себе базовую логику отправки запросов. В конструктор передается базовый
‘адрес сервера и опциональный объект с заголовками запросов. Методы:

- `get` - выполняет СЕТ запрос на переданный в параметрах ендпоинт и возвращает
промис с объектом, которым ответил сервер

- `post` - принимает объект с данными, которые будут переданы в JSON в теле
запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове
метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть
переопределен заданием третьего параметра при вызове.

#### Класс EventEmitter
Брокер событий позволяет отправлять события и подписываться на события,
происходящие в системе. Класс используется в презентере для обработки событий ив
слоях приложения для генерации событий.

Основные методы, реализуемые классом описаны интерфейсом `IEvents`:
- `on` - подписка на событие,
- `emit` - инициализация события, 
- `trigger` - возвращает функции, при вызове которой инициализируется требуемое в параметрах событие.

### Слой данных 
#### Класс `Card` 
Класс предназначен для обработки логики с карточками

Поля:
- card: ICard - карточка

Методы:
- addBasket(cardId: string): void - добавляет товар в корзину
- deleteCard(cardId: string): void - удаляет выбранный товар из корзины
- checkBasket(cardId: string): boolean - проверяет есть ли товар в корзине, чтобы определить добавить или удалить товар


#### Класс `CardsData` 
Класс отвечает за хранение и логику работы с данными карточек. \
Конструктор класса принимает инстанс брокера событий.

В полях класса хранятся следующие данные: 
- _cards: ICards[]  - массив объектов карточек 
- _preview: string | null - id карточек, выбранной для просмотра в модальном окне
- events: IEvents - экземпляр класса `EventEmitter` для инициализации событий при изменении данных. 

Так же класс предоставляет набор методов для взаимодействия с этими данными.
- getCard(cardId: string): ICard - возвращает карточку по ее id 
- а также сеттеры и геттеры для сохранения и получения данных из полей класса 

<br>

#### Класс `OrderData` 
Класс отвечает за хранение и логику с данными текущего пользователя. \
Конструктор класса принимает инстанс брокера событий. 

В полях класса хранятся следующие данные: 
- email: string - почта пользователя
- phone: number - телефон пользователя
- payment: string - способ оплаты
- address: string - адрес 
- events: IEvents - экземпляр класса `EventEmitter` для инициализации событий при изменении данных. 

Так же класс предоставляет набор методов для взаимодействия с этими данными.

- setUserInfo(orderData: IOrderForm): void - сохраняет данные пользователя в классе
- checkUserValidation(data: Record<keyof UserInfo, string>): boolean - проверяет объект с данным пользователя на валидность.

<br>

#### Класс `Basket` 
Класс отвечает за хранение и логику с корзиной товаров. \
Конструктор класса принимает инстанс брокера событий. 

В полях класса хранятся следующие данные: 
- cards: ICard[] - массив товаров в корзине 
- events: IEvents - экземпляр класса `EventEmitter` для инициализации событий при изменении данных. 


Метод: 
- total(cards: ICard[]): number - итоговая стоимость всех товаров в корзине

<br>

### Слой представления 
Все классы представления отвечают за отображение внутри контейнера(DOM-элемент) передаваемых в низ данных. 

#### Класс `Modal` 
Реализует модальное окно. Представляет методы `open` и `close` для управления отображением модального окна. Устанавливает слушатели на клавиатуру, для закрытия модального окна по Esc, на клик в оверлей и кнопку-крестик для закрытия попапа.

- constructor(selector: string, events: IEvents) - Конструктор принимает селектор, по которому в разметке страницы будет идентифицировано модальное окно и экземпляр класса `EventEmitter` для возможности инициации событий.

Поля класса 
- modal: HTMLElement - элемент модального окна
- events: IEvents - брокер событий

<br>

#### Класс `ModalWithConfirm`
Расширяет класс `Modal`. Предназначен для реализации модального окна с тем, что заказ оформлен.

Поля класса: 
- submitButton: HTMLButtonElement - кнопка подтверждения

Методы: 
- total(cards: ICards): number - количество которое с вас списано
- get form: HTMLElement - геттер для получения элемента формы

<br>

#### Класс `ModalWithForm`
Расширяет класс `Modal`. Предназначен для реализации модального окна с формой содержащей поля ввода. При сабмите инициирует событие передавая в него объект с данными из полей ввода формы. Предоставляет методы для отображения ошибок и управления активностью кнопки.

Поля класса :
- submitButton: HTMLButtonElement - кнопка продолжения
- _form: HTMLFormElement - элемент формы 
- formName: string - значение атрибута name формы
- inputs: NodeListOf<HTMLInputElement> - коллекция всех полей ввода формы
- errors: Record<string, HTMLElement> - объект хранящий все элементы для вывода ошибок под полями формы с привязкой к атрибуту name инпутов.

Методы:
- setValid(isValid: boolean): void - изменяет активность кнопки подтверждения
- showInputError(field: string, errorMessage: string) - отображает полученный текст ошибки под указанным полем
- hideInputError(field: string): void - скрывает текст ошибки под указанным полем ввода 
- close(): void - расширяет родительский метод при закрытии очищая поля формы и деактивируя кнопку сохранения
- get form: HTMLElement - геттер для получения элемента формы

<br>

#### Класс `ModalWithBasket`
Расширяет класс `Modal`. Предназначен для реализации модального окна с корзиной. Выводит массив товаров, которые пользователь добавил в корзину, также при желании их можно удалить.

Поля:
- submitButton: HTMLButtonElement - кнопка продолжения
- _item: HTMLElement - разметка товара в корзине


Метод:
- switcherCard(card: HTMLElement): void - добавляет/удаляет разметку товара
- get form: HTMLElement - геттер для получения элемента формы


<br>

#### Класс `ModalWithCard`
Расширяет класс `Modal`. Предназначен для реализации модального окна с информацией о товаре. Содержит кнопку по которой товар можно добавить или убрать из корзины.

Поля: 
- info: ICard - информация о товаре

Метод:
- switchButton(button: HTMLButtonElement): void - принимает разметку кнопки и меняет ее название

<br>

#### Класс `Card`
Отвечает за отображение карточки, задавая в карточке данные названия, изображения, цены, id, категории товара. Класс используется для отображения карточек на странице сайта. В конструкторе класса предается DOM элемент темплейта, что позволяет при необходимости формировать карточки разных вариантов верстки. В классе устанавливаются слушатели на каждую карточку, при нажатии на которую пользователь может открыть ее.

Поля класса содержат элементы разметки элементов карточки. Конструктор, кроме темплейта принимает экземпляр `EventEmitter` для инициализации событий.

Методы:
- setData(cardData: ICard): void - заполняет атрибуты элементов карточки данными
- render(): HTMLElement - метод возвращает полностью заполненную карточку с установленными слушателями
- геттер id - возвращает уникальный id карточки 
 
<br>

#### Класс `CardsContainer`
Отвечает за отображение блока с карточками на главной станице. \
Предоставляет метод `addCard(cardElement: HTMLElement)` для отображения карточек на страницу. В конструктор принимает контейнер, в котором размещаются карточки.

## Взаимодействие компонентов 
Код, описывающий представления и данных между собой находится в файле `index.ts`, выполняющем роль презентера. \
Взаимодействие осуществляется за счет событий генерируемых с помощью брокера событий и обработчиков этих событий, описанных в `index.ts`. \
В `index.ts` сначала создаются экземпляры всех необходимых классов, а затем настраивается обработка событий.

*Список всех событий, которые могут генерироваться в системе:*
- `order:input` - ввод в данных в форме со способом оплаты и адресом 
- `order:submit` - сохранение данных пользователя в модальном окне
- `order:validation` - событие, сообщающее о необходимости валидации формы со способом оплаты и адресом 
- `contacts:input` - ввод в данных в форме с контактами
- `contacts:submit` - сохранение данных пользователя в форме
- `contacts:validation` - событие, сообщающее о необходимости валидации формы с контактами
- `basket:button` - кнопка оформления к корзине товаров
- `basket:delete` - кнопка удаления товара из корзины
- `basket:total` - итоговая стоимость всех товаров в корзине
- `card:select` - выбор карточки для отображения в модальном окне
- `card:add` - помещение товара в корзину
- `card:delete` - удаление товара из корзины
- `order-success:submit` - закрытие модального окна об успешном заказе
